# FPL Language Roadmap

## Phase 1: Foundation (current)

- Specification and EBNF grammar
- Initial examples
- Conformance fixture layout
- Experimental Go reference parser scaffold

## Phase 2: Language tooling

- Complete AST for rules, conditions, and literals
- Stable parser diagnostics and error codes
- fplfmt formatting rules and idempotent formatter
- Fuzz tests for lexer/parser

## Phase 3: Ecosystem

- VS Code grammar package
- Tree-sitter grammar
- Conformance runner that executes implementations against fixtures
- Versioned language compatibility matrix

## Phase 4: Multi-implementation maturity

- Secondary parser implementation (TypeScript or Rust)
- Cross-implementation conformance parity
- RFC process for language evolution
